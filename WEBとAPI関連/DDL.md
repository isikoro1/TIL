# DDL とは
「データ定義言語」（Data Definition Language）の略
データベースの構造（テーブル、インデックス、ビューなど）を
定義、変更、削除するために使用されるSQLのコマンド群

主なコマンドには、オブジェクトを「作成」（CREATE）、
既存のものを「変更」（ALTER）、
削除（DROP）するもの、
そしてテーブル内のデータをすべて消去する（TRUNCATE）もの

CREATE: 新しいデータベース、テーブル、ビュー、インデックスなどを作成します。
ALTER: 既存のテーブルのカラム定義の追加、変更、削除など、構造を変更します。
DROP: 既存のデータベース、テーブル、ビューなどを削除します。
TRUNCATE: テーブル内のすべてのデータを削除しますが、
テーブルの構造自体は残します。

DDLと他のSQL言語との比較
DML（Data Manipulation Language：データ操作言語）: 
テーブルに格納された個々のデータを操作（挿入、更新、削除、検索など）する言語です。
DCL（Data Control Language：データ制御言語）: 
データベースへのアクセス権限を管理する言語（GRANT、REVOKEなど）。
